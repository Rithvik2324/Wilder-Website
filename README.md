# Wilder Belize Adventures

A modern, fully responsive marketing & booking-inquiry website for **Wilder Belize Adventures**, a family-run adventure tour operator in Placencia, Belize. Built as a complete rebuild of the old `wilderbelizeadventures.com` with a vibrant tropical-adventure design.

## Tech stack

- **Next.js 16** (App Router) + **React 19**
- **Tailwind CSS v4** (CSS-based design tokens in `src/app/globals.css`)
- **Framer Motion** for scroll reveals, parallax, and micro-interactions
- **lucide-react** icons
- Minimal backend: a single API route (`/api/inquiry`) that saves form submissions to a local JSON file

## Getting started

```bash
npm install
npm run dev      # http://localhost:3000
```

Build for production:

```bash
npm run build
npm start
```

## Features

- **Cinematic homepage** — parallax/Ken-Burns hero, featured tours, value props, categories, stats, testimonials, interest filters, transfers teaser.
- **Tours catalog** (`/tours`) — live client-side filtering by category, difficulty, interest and price, with sorting and a mobile filter drawer.
- **Tour detail pages** (`/tours/[slug]`) — 23 real tours with hero, photo gallery + lightbox, highlights, itinerary timeline, what's included / to bring, meeting-point map, related tours, and a sticky booking widget (with a mobile booking bar).
- **About, Travelers Info, Transfers, Contact** pages.
- **Booking / contact / newsletter forms** — validated, with a honeypot and saved server-side. Tour bookings proceed to secure payment through Belize Bank.
- **SEO** — per-page metadata, OpenGraph/Twitter cards, JSON-LD (`TravelAgency`, `TouristTrip`, `BreadcrumbList`), `sitemap.xml`, `robots.txt`.
- Accessible (keyboard focus rings, labelled controls, AA-contrast CTAs) and fully responsive.

## Payments

Tour bookings use Belize Bank's hosted payment flow. The booking widget directs customers to payment and confirms the booking after a successful payment.

## Form submissions

Submissions are appended to `data/submissions.json` (git-ignored) by `src/app/api/inquiry/route.ts`. This is a deliberately minimal store; swap it for a database or email service for production.

## Content & images (how they were generated)

- Tour content and site copy live in `src/data/tours.ts` and `src/data/site.ts`. They are generated by `scripts/build-data.mjs` (re-run with `node scripts/build-data.mjs` if you edit the source data).
- Photos in `public/images/` are free-to-use stock (Pexels) downloaded via `scripts/download-images.mjs`. Replace them with the operator's real photography for launch.
